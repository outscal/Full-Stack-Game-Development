While programming any in-game entity, there are situations when they need to behave differently given some conditions are satisfied.



Lets take a look at a popular example, our friend **Mario**:



In Mario, when a certain power up is consumed, behavior of Mario is changed with respect to its surrounding objects and enemies.



![](https://outscal-assets.s3.ap-south-1.amazonaws.com/qa2/LMS/628efc0ffce47713f7782ce7/10_13_2023__13_20_16.png)



You might be thinking that conditional statements can be useful for implementing the above behavior. But now imagine a game like Skyrim or GTA, where players and enemies might be having hundreds of **Behavior's**.



If you choose to employ a brute-force method in this case, the code will quickly become a spaghetti bowl of if-else conditions, nested to the moon and back.



To solve this problem like a pro game developer, you can implement the State Design Pattern. That is what this course is all about.

In this course we are going to make a clone of the popular mobile game known as **Hunter Assassin**. We are going to learn about State Machines and use it to implement a variety of enemy behavior. 



State Machine is widely used in the Gaming Industry for implementing enemy AI. Do you recall any Boss Fights where the enemy displayed a set of behaviors being executed in a certain sequence? Or maybe the enemy changed its attack combos when its health dropped below a certain point? That might be the working of State Machines behind the scenes 😉



At the end of this course you will have a complete playable game project with generic state machine implemented. You will be creating a scalable system for Enemy AI which you may use later to create your own Enemy Behaviors.
