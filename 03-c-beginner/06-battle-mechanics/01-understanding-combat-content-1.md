## Randomizing Attack Damage and Heal


---

Currently, the attack damage and the healing capacity are constant. 

Imagine *Dough Master* always giving only 20 damage. So Boring🥱



We will add a random number to the attack damage and healing capacity.

This will make the attack and healing less tedious and more unpredictable!



The `generateRandomNumberInRange(min, max)` function will generate this random number.



But how to generate a random number in C#?🤔



> **PROTIP💡: Random class**
> `Random` is a class in the `System` namespace.
> `Next(min, max+1)` function of the `Random` class is used to generate a random number between a specified range.



In the `generateRandomNumberInRange()` you need to initialize the object of the `Random` class,

```csharp
Random rand = new Random();
```


then use the `Next(min, max+1)` function to generate a random number by passing the minimum and maximum numbers as parameters.

Lastly, the random number generated will be returned!

```csharp
int randomNumber = rand.Next(min, max + 1);
return randomNumber;
```



**Note: **In the `Next` function, you must pass `max+1` as the maximum number as the Next function doesn't include the maximum number passed.

`**main.cs**`** **`**Player class**`

```csharp
private int generateRandomNumberInRange(int min, int max)
{
        Random rand = new Random();
        int randomNumber = rand.Next(min, max + 1);
        return randomNumber;
 }
```

# Flaws in Letho


---

Have a look at the `Take Damage` function of *Letho*:



![ ](//outscal.github.io/Full-Stack-Game-Development/images/0f45e55f59338cb1.png)



What do you think is the problem here?🤔

The role of the `TakeDamage()` ideally should be to reduce the player’s health by a certain amount of damage.

But in this code `TakeDamage` is handling three responsibilities

- Reducing the health
- Showing the health after damage
- Displaying game loss screen

This **violates **the **SRP**!



> **PROTIP 💡: SRP**
> Every class or method should have **one specific responsibility**. This keeps your code modular, easier to maintain, and more scalable.



This was not only in `TakeDamage` but in most of the functions.
