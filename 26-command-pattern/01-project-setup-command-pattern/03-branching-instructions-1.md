The default branch `*(main)*`* *of this project is your base branch.



As you progress in this module while adding new game features, you will create separate `**feature branches**` extending from this `**main**` branch.



On these **feature branches** you create, you will push your commits. 

All the assignments must be done in these feature branches.



Once you are done with any of your assignment, You will create a Pull Request from the feature branch to the main branch and submit the link of this PR (Pull request) as submission for that assignment.



Take a look at the below diagram for reference:

![branching instructions](//outscal.github.io/Full-Stack-Game-Development/images/44d74d2411c8d7e2.png)



**Please make sure that while creating a Pull Request, ****your target branch is of your own forked repo****. **

**The PR must not be made for the original repo provided to you by Outscal.** 



Whenever you are done implementing any new feature in your game, we will let you know.

At that point, you can go to the provided **solution branch** and compare your changes with the code in solution branch.

The **solution branch** is provided for self-evaluation of your code, once you are done with the guided assignments.

The **solution branch** provides an ideal solution for the assignments given to you.

Once you have self-evaluated your code with the solution branch and all seems fine, you can go ahead and merge your PR from `feature_1`.

For the next feature, you should create another branch called `feature_2` from the `main` branch



You can spoil yourself by taking a look at the solution before trying out the assignments yourself, but Outscal does not recommend you to learn that way.

If you feel you are stuck somewhere, we recommend you to engage with our discord community of thousands of game developers. It is a very helpful and inviting place where you will find a lot of people with similar interest as you.
