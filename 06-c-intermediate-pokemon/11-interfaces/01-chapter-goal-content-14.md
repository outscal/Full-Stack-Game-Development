By the end of this chapter, you’ll know how to build something called an **interface**.


Interfaces are useful for defining a set of rules or behaviors that other classes must follow without worrying about how those behaviors are implemented.



> **Interface :**
> an interface is a class that only contains pure virtual functions. It doesn’t provide any implementation of the functions, just the declarations. 
> Other classes that inherit from this interface must provide the actual code for these functions.





You’ll make an interface called `IStatusEffect`.

This will help you create effects like Burn and Paralysis.
Which you’ll be able to add to your battle system.


It won’t just be for this game—learning about interfaces will help you in many coding projects in the future.


Interfaces are powerful because they let you write flexible code.
Once you understand them, your code will be easier to update and reuse.
