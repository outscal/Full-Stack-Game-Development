The purpose of a code review is to make you aware of your mistakes and give you direction on how to fix them.



A good code review should

- Is the one in which a lot of your mistakes are being pointed out.
- Not shy away from pointing out your mistakes
- Provide reasoning as to why what you have done is wrong
- Should provide feedback on how to improve your mistakes
- Takes notice of the code architecture and helps you in improving it.
- Check for the ReadMe file of your GitHub Repository to evaluate how it is being represented.

We expect you to raise a red flag if:

- you feel like your code reviews are not up to the mark.
- If any of the given above pointers are not being followed, 

- If you feel like your code review is being rushed or mistakes are not being pointed out clearly.



Once your code review is done, you will get a feedback form to fill. Please raise any issues regarding code review quality in it.
