Getting rejected in code reviews is pretty common because of the standards that Outscal maintains. 

Working hard for something and getting rejected in the end can be naturally demotivating or frustrating, but **it is necessary**.



Handling rejection in code reviews is a crucial aspect of growth as a game developer. 

Rather than feeling demotivated, view rejection as an opportunity for improvement. 



Embrace it as feedback that helps refine your skills and deepens your understanding. 

Each rejection is a chance to learn, iterate, and ultimately produce better work. 



So, welcome rejections knowing that **each setback will bring you closer to success**.
