Conflict resolution occurs when Git encounters conflicting changes between the branches being merged.

Conflicts can arise when two or more branches modify the same part of a file independently.



![placeholder](/Full-Stack-Game-Development/images/ce5bafe4b316f9e4.jpg)





GitHub Desktop provides tools to help you resolve conflicts. When conflicts occur during a merge, GitHub Desktop will display a notification and prompt you to resolve the conflicts manually.



![placeholder](/Full-Stack-Game-Development/images/4c82406f50222c93.png)





You can open the conflicted files in an external editor to resolve the conflicts, choose which changes to keep, and then mark the conflicts as resolved.

Once conflicts are resolved, you can commit the changes to finalize the merge operation.



To learn more about resolving conflicts, visit the official GitHub Desktop docs:

https://docs.github.com/en/desktop/making-changes-in-a-branch/committing-and-reviewing-changes-to-your-project-in-github-desktop
