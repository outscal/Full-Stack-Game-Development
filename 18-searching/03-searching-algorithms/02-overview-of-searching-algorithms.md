**Searching algorithms** are all about finding specific elements within a set of data.

Their main job is to check whether the element you're looking for is present in the dataset. 

If the element exists, the algorithm will find its exact location and give it to you.



These algorithms ensure the search process is quick, whether it's for databases, inventories, or game navigation.



Before diving into Search algorithms, you need to understand the following :-

- **Target Element**
- **Search Space**
- **Time & Space Complexity**



Let's understand them individually.



## **Target Element**



Every search operation revolves around a specific **target element** within a set of data collection. 

This can be anything from a value or record to a key or any other data of interest.



For example:

In **PacMan**, you are the target element for the ghosts present in the game. 

The ghosts keep searching the maze to find and catch you while you try to run away from them.



![ ](//outscal.github.io/Full-Stack-Game-Development/images/601cb1d6bb6d1b39.gif)





## **Search Space**



The search space refers to the entire collection of data you're searching through to find the target element. 

How this data is organized depends on the data structure used, like an array or a linked list. 

The search space may vary in size and organization depending on the data structure used. 



*In chess, the ****whole Chessboard**** is the ****space search****.*

![  ](//outscal.github.io/Full-Stack-Game-Development/images/82893df32a46d1d6.png)

*Search space = Chessboard*





## **Time & Space Complexity**



Different search algorithms take varying amounts of time and space to find the target. 



**Time Complexity:** how long it takes the algorithm to complete the search, which is expressed in terms of the size of the data it's searching through.

**Space Complexity:** how much additional memory the algorithm needs to perform the search.
