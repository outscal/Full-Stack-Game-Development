<h3>Things to remember when you receive an assignment from the company</h3>
<h4>All these points are super important, you have to submit each and every assignment in this way only.</h4>
<ol>
<li>You have to <b>always</b> submit your <b>git repository</b> as a submission with additional resources(videos, builds) if asked.</li>
<li>Even if the company is not asking you to <b>make a build and video.</b></li>
<li><b>Do not</b> rush just because there is a deadline.</li>
<li><b>Do not</b> write shitty code.</li>
<li><b>Do not copy-paste</b> the solution from any online repositories / do not copy-paste the entire tutorial on youtube as it is. If you are doing this, you will be blacklisted from Outscal’s placement process as well as you will be rejected by the company immediately.</li>
<li>You just shouldn't be submitting the implementation but you should submit the assignment with the <b>best quality of code that you have ever written.</b></li>
<li>Use each and every design pattern that you have learnt till now in the assignment</li>
<ol>
<li><b>MVC →</b> Don't implement MVC for the entire project but at least create MVC architecture for Player or any other important entity.</li>
<li><b>State Machine →</b> Identify where you can make states in the problem statement and implement state machine design patterns to enhance your code architecture.</li>
<li><b>Observer Pattern →</b> Spot the exact use case of the observer pattern and write proper events in the assignment that you are submitting to showcase your observer pattern as a skill set.</li>
<li><b>Singleton →</b> This is the most common design pattern that you can use, but you need to use this pattern carefully, excessive use of singleton design patterns can make your code architecture loosely coupled which can lead to bugs.</li>
<li><b>Object Pooling/Factory Design Pattern →</b> The most important design pattern that you have to use for optimizing your assignment. Wherever you are re-using the same game objects, again and again, you can implement object pooling.</li>
</ol>
<li>Always follow SRP(Single Responsibility Principle) in your code architecture of assignment.</li>
<li>Make sure to use enums to increase the readability of your code.</li>
<li>Maintain <b>a proper folder hierarchy</b> in your solution repository where each and every type of script are well organized.</li>
<li>Add a <b>proper ReadMe file</b> with useful screenshots and a description of the features and design patterns that you have implemented in the solution.</li>
</ol>
