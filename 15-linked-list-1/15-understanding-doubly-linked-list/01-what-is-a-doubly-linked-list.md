## Understanding Doubly Linked LIst


---

So what's the big deal about Doubly Linked List?

## **Imagine riding a metro train 🚇**



In a singly linked list, you can only move forward, imagine an Indian railway train that only goes one way.

If you miss your stop, you have to go back to the beginning and start over. Frustrating, right?



But with a doubly linked list, it is like a metro train that can move in both directions. You can easily go back to a previous stop without any worry. 

Each train cart (node) is connected to the ones before and after it, allowing smooth travel in either direction.

***Train as a Doubly Linked List***



Like a metro system, a doubly linked list makes it easy to traverse your data structure in both directions, allowing for more flexible operations.

**Understood??**



Let me explain the same thing but in a technical way:

A **Doubly Linked List** is a complex type of linked list in which each node contains three components:

- **Data**: The essential information or value held by the node.
- **Next Pointer**: A reference to the next node in the sequence, facilitating traversal of the list forward.
- **Previous Pointer**: A reference to the previous node in the sequence, allowing backward traversal of the list.

> **So if a Linked List ⇒  **`**A → B → C**`
> **Then a Doubly Linked List ⇒  **`**A ⇆ B ⇆ C**`
