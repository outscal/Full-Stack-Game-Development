As discussed in the chapter goal, this isn't very different from the sorts you have covered so far.

The time and space complexity is also similar to them.



## TIME COMPLEXITY


---

Selection Sort works by repeatedly finding the minimum element from the unsorted part of the list and moving it to the sorted part.

In all scenarios (best, average or worst), the time complexity which is `**O(n²)**` will remain the same as Selection Sort compare each element with every other element to find the minimum.



## SPACE COMPLEXITY


---

Similar to Bubble and Insertion sort, Selection sort is also an in-place sorting algorithm, which means it doesn't require any additional memory for sorting.



So, the space complexity is  `**O(1)**`.




## CHARACTERISTICS OF Selection SORT


---

- **Simple Implementation:** Just like Bubble and Insertion sort, Selection sort is easy to implement and understand, making it the basics of sorting algorithms.
- **Stable with Modification:** The basic implementation of selection sort is not stable. However, it can be made stable with a few modifications, such as swapping with the next position to be filled only if the selected item is not already in that position.
- **Not Adaptive: **Unlike some other sorting algorithms like insertion sort, selection sort makes no significant performance gains from an already sorted or nearly sorted array.



## **APPLICATIONS**


---

- **Small Data Sets**: Selection Sort is for sorting small data sets where simplicity and ease of implementation are prioritized over efficiency.
- **Memory Constraints:** In environments with extremely limited memory space, the constant space complexity of Selection sort can be a significant advantage.
